啟動monitor docker:

docker compose -f docker-compose.monitoring.yml up -d

啟動後，Prometheus 將會在 http://localhost:9091 上運行，Grafana 則在 http://localhost:3001。

請依照以下步驟進行驗證：

開啟 Grafana：

在瀏覽器中開啟 http://localhost:3001
登入系統：

使用者名稱：admin
密碼：admin
檢查資料來源：

登入後，點擊左側選單的 齒輪圖示 (Configuration) > Data Sources。
您應該會看到一個名為 "Prometheus" 的資料來源，並且狀態正常。
檢查儀表板：

點擊左側選單的 四個方塊圖示 (Dashboards)。
您應該會看到一個名為 "TixMaster Overview" 的儀表板。點擊進入。
請確認您是否能成功登入，並看到有數據（或至少有圖表框架）的儀表板。完成後請告訴我結果。

# 指令
1. 停止並移除目前的監控容器：

docker-compose -f docker-compose.monitoring.yml down
2. 刪除 Grafana 的資料卷： (注意：Docker Compose 通常會以專案資料夾的名稱作為前綴，所以 volume 的名稱會是 TixMaster_grafana-data)

docker volume rm TixMaster_grafana-data
如果提示找不到該 volume，您可以執行 docker volume ls 來確認一下確切的名稱。

3. 重新啟動監控服務：

docker-compose -f docker-compose.monitoring.yml up -d
完成後，請再次嘗試開啟 Grafana (http://localhost:3001)，看看這次是否能成功載入。
